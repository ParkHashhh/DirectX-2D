
로컬공간 : 모델 자체 기준 좌표. 메시를 제작할 때 사용하는 좌표이다.

월드공간 : 게임이 동작하는 공간. 행렬을 이용해서 로컬공간의 물체를
월드공간으로 변환한다. 위치, 크기, 회전 정보를 포함할 수 있다.
위치, 크기, 회전은 행렬로 표현할 수 있다.
회전은 X축, Y축, Z축 회전을 조합하여 표현도 가능하다.
월드행렬 = 크기행렬 * 자전행렬 * 이동행렬 * 공전행렬 * 부모행렬

뷰공간 : 카메라 기준 좌표계. 월드공간에 배치되어 있는 카메라를 
원점(0, 0, 0)으로 이동시키는 만큼 모든 물체를 이동시키고 카메라가 Z+ 방향
을 바라볼 수 있게 회전되는 만큼 모든 물체를 원점을 중심으로 회전시켜
카메라 중심의 공간을 구성하는 방식이다. 행렬로 월드공간의 물체를 뷰공간으로
변환한다.

투영공간 : 3D를 2D로 변환하는 과정. 행렬로 뷰공간의 물체를 투영공간으로
변환한다. 뷰공간의 정점정보를 x, y를 -1 ~ 1, z를 0 ~ 1 사이로 
변환하게 된다. 원근감을 적용해서 화면에 출력하기 위한 과정으로
원근투영, 직교투영이 있다.
원근투영 : 원근감을 적용하는 투영
직교투영 : 원본 크기로만 적용하는 투영
x, y -1 ~ 1, z 0 ~ 1 사이를 벗어나게 되면 클리핑으로 짤리게 된다.

===== DX가 알아서 처리하는 과정 =====
동차공간(Clip / NDC Space) : GPU가 실제로 사용하는 정규화 좌표.
투영공간으로 변환이 일어나면 x, y는 -1 ~ 1 사이, z는 0 ~ 1 사이로
변환이 되게 된다. w에는 뷰공간에서의 z값이 들어가 있다.
x, y, z, w 가 있을 경우 w로 나누어주게 된다.
x / w, y / w, z / w

뷰포트 변환 : DirectX를 초기화할 때 지정한 해상도의 2D(Screen Space)공간
으로 변환한다. 1280, 720의 해상도를 가지고 있을 경우 0 ~ 1280, 0 ~ 720의
Screen Space로 변환하게 된다. 실제 화면좌표이다.
